Section 1

Introduction

About the App

Stars SRL Project is a web-based application designed to facilitate project management and collaboration for students and teachers. The app allows users to create, edit, and manage project briefs, which outline the requirements and expectations for various assignments.

Assigned Flows.

For this report, I have chosen to analyze two key flows within the Stars SRL Project app: Creating a Brief and Editing a Brief. These flows are essential for both students and teachers as they involve the core functionality of defining and updating project requirements. By examining these flows, I aim to identify pain points and areas for improvement in the user experience.

Why these flows?

These flows were chosen because they represent fundamental interactions that users have with the app, and they directly impact the efficiency and effectiveness of project management. Understanding the user journey through these flows will help in identifying opportunities for enhancing the overall user experience.

Top 3 Pain Points

  1. Saving Changes - The save button is hard to find, which can lead to users losing their work if they cannot save their changes. This violates the error prevention heuristic.
  2. Locating the Create/Edit Buttons - The buttons for creating and editing briefs can be hard to spot, which can hinder users from starting or modifying their briefs effectively. This violates the consistency and standards heuristic.
  3. Updating Assignment Aspects - The process for updating assignment aspects is not intuitive, making it difficult for users to modify assessment criteria as needed. This violates the recognition rather than recall heuristic.

Design Briefs

Design Brief 1

Issue: Users have difficulty finding the save button, which can lead to loss of work.

Brief: Redesign the interface to make the save button more prominent and easily accessible. This could be as simple as changing its color or size.

Design Brief 2

Issue: The create and edit buttons for briefs are not easily visible.

Brief: Redesign the interface to make the create and edit buttons more prominent and easily accessible. This could be as simple as changing their color or size.

Design Brief 3

Issue: The process for updating assignment aspects is not intuitive.

Brief: Redesign the interface to make the update process more intuitive and user-friendly. This could involve simplifying the workflow or providing clearer instructions.
